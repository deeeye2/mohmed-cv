About DevOps Bot

DevOps Bot is a powerful tool designed to assist with various DevOps tasks, making it easier to manage infrastructure and deployments. It offers functionalities for creating Kubernetes manifests, Dockerfiles, Ansible playbooks, and Terraform configurations.

DevOps Bot is written in Python and uses Flask for its web framework. The backend process involves handling HTTP requests, generating configuration files based on user input, and interacting with a knowledge base for issue resolution.

Features

  • Greet: A simple greeting command.
  • Version: Displays the current version of DevOps Bot.
  • mkdir: Creates a directory at the specified path.
  • solve: Provides solutions to common issues using the knowledge base.
  • login: Authenticates the user and retrieves an access token.
  • create: Generates configuration files for Kubernetes, Docker, Ansible, and Terraform.

Backend Process

The backend of DevOps Bot works as follows:

  1. The user interacts with the CLI, sending commands and parameters.
  2. The CLI sends HTTP requests to the Flask-based backend server.
  3. The server processes the requests, generating the required configuration files or providing solutions from the knowledge base.
  4. The server responds with the generated files or the solution, which is displayed to the user via the CLI.

System Requirements

Ensure your system meets the following requirements:

  • Memory: 2GB RAM (4GB recommended)
  • CPU: 2 cores (4 cores recommended)
  • Disk Space: 1GB (5GB recommended)
  • OS: Ubuntu 20.04 or higher

Installation Steps

1. Update and install required packages:
   sudo apt-get update
   sudo apt-get install -y python3 python3-venv python3-pip git

2. Clone the repository:
   git clone https://github.com/deeeye2/devops-bot.git ~/devops_bot

3. Navigate to the installation directory:
   cd ~/devops_bot

4. Create a virtual environment:
   python3 -m venv env

5. Activate the virtual environment:
   source env/bin/activate

6. Upgrade pip and install wheel:
   pip install --upgrade pip
   pip install wheel

7. Install the devops-bot package:
   pip install .

8. Create the systemd service file:
   sudo tee /etc/systemd/system/devops_bot.service > /dev/null <
            
Uninstallation Guide

            
1. Stop the DevOps Bot Service:
   sudo systemctl stop devops_bot.service

2. Disable the DevOps Bot Service:
   sudo systemctl disable devops_bot.service

3. Remove the systemd service file:
   sudo rm /etc/systemd/system/devops_bot.service

4. Reload systemd:
   sudo systemctl daemon-reload

5. Deactivate the virtual environment if active:
   deactivate

6. Remove the DevOps Bot directory:
   rm -rf ~/devops_bot

7. Remove the symbolic link:
   sudo rm /usr/local/bin/devops-bot

8. Optionally remove installed Python packages:
   rm -rf ~/devops_bot/env

9. Optionally remove system-wide dependencies (be cautious):
   sudo apt-get remove --purge -y python3 python3-venv python3-pip git
   sudo apt-get autoremove -y

echo "DevOps Bot uninstalled successfully."
